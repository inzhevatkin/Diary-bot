# Telegram Diary Bot

Личный Telegram-бот для дневника питания и самочувствия. Бот принимает текст, голосовые сообщения и фотографии, конспектирует записи и сохраняет их локально в `data/diary.jsonl`. Также он может раз в день запускать короткий чек-ин по вопросам из `questions.json`.

## Приватность

Проект рассчитан на локальное хранение личных данных. По умолчанию `.gitignore` исключает из публикации:

- `.env` и локальные варианты файлов окружения с токенами и API-ключами;
- `data/` с дневником, фото и голосовыми сообщениями;
- рабочие контекстные заметки проекта;
- виртуальное окружение и кеши Python.

Перед публикацией своего репозитория убедитесь, что в git попали только код, пример настроек и обезличенная документация.

## Быстрый старт

1. Создайте бота через BotFather и получите токен.
2. Скопируйте `.env.example` в `.env`.
3. Впишите `TELEGRAM_BOT_TOKEN`.
4. Установите зависимости:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

AI-функции опциональны. Если они нужны, установите их отдельно:

```powershell
# Конспектирование текста через GigaChat
pip install -r requirements-gigachat.txt

# Расшифровка голоса и описание фото через OpenAI
pip install -r requirements-openai.txt
```

5. Запустите:

```powershell
python bot.py
```

6. Напишите боту `/start`. Он покажет ID чата. Впишите его в `.env` в `ALLOWED_CHAT_IDS`, чтобы дневник был личным.

## Как это работает

- Текстовые сообщения сразу записываются в дневник.
- Голосовые сообщения сохраняются в `data/uploads` и расшифровываются, если задан рабочий `OPENAI_API_KEY`.
- Фотографии сохраняются в `data/uploads` и описываются, если задан рабочий `OPENAI_API_KEY`.
- Вопросы для ежедневной проверки лежат в `questions.json`.
- Записи хранятся построчно в JSONL: одна строка равна одной записи.
- Для каждой записи сохраняются технические даты отправки и обработки (`message_sent_at`, `received_at`) и смысловая дата дневника (`diary_date`). Команда `/today` использует `diary_date`.
- Дневниковый день может начинаться не в полночь. По умолчанию `DIARY_DAY_START=04:00`, поэтому записи, отправленные после полуночи, но до 04:00, относятся к предыдущему дневниковому дню.

## Просмотр в браузере

Локальный просмотрщик читает `data/diary.jsonl` и показывает дневник в браузере:

```powershell
python viewer.py
```

После запуска откройте:

```text
http://127.0.0.1:8000
```

Страница показывает записи по дням, чек-ины, разгрузочные дни, поиск и фильтр по типам записей. Также на странице есть график боли по дням, таблица чек-инов и блок возможных пищевых триггеров. Триггеры считаются как совпадения между группами продуктов и уровнем боли в тот же день, на следующий день и через 2 дня; это не медицинский вывод, а подсказка для проверки гипотез. По строке триггера можно кликнуть, чтобы увидеть конкретные дни и записи, где эта группа была найдена. Просмотрщик работает локально и по умолчанию слушает только `127.0.0.1`.

Просмотрщик также умеет читать будущий импорт данных браслета из файла:

```text
data/imports/health_connect.json
```

Минимальный формат:

```json
[
  {
    "date": "2026-06-17",
    "steps": 8200,
    "sleep_minutes": 430,
    "resting_heart_rate": 62,
    "average_heart_rate": 78,
    "active_minutes": 45,
    "calories": 2100
  }
]
```

Эти данные показываются в таблице "Данные браслета" рядом с болью за тот же день.

## Команды

- `/start` - показать приветствие и ID чата.
- `/help` - показать список доступных команд.
- `/last` - показать последнюю запись дневника.
- `/today` - показать все записи за сегодня.
- `/day` - показать записи за выбранный дневниковый день.
- `/add` - следующую текстовую запись сохранить за выбранную дату.
- `/cancel` - отменить активный режим ввода.
- `/delete_last` - удалить последнюю запись текущего чата с созданием резервной копии.
- `/checkin` - вручную запустить ежедневную проверку самочувствия.
- `/fasting` - отметить текущий дневниковый день как разгрузочный, когда еды не было.
- `/export` - отправить дневник Markdown-файлом.

Команды `/day`, `/add` и `/fasting` принимают дату:

```text
/day вчера
/day сегодня
/day 2026-06-05
/add вчера
/add сегодня
/add 2026-06-05
/fasting вчера
/fasting сегодня
/fasting 2026-06-05
```

После `/add 2026-06-05` бот попросит прислать текст. Следующее текстовое сообщение будет сохранено как обычная дневниковая запись, но с `diary_date`, равной выбранной дате. Техническая дата отправки сообщения сохранится отдельно.

Команда `/fasting` сохраняет отдельную запись типа `fasting_day`, чтобы разгрузочный день был виден в дневнике и не выглядел как случайно пропущенный день.

Команда `/delete_last` удаляет последнюю запись именно текущего чата и перед перезаписью дневника создает backup в `data/backups/`.

Перед ежедневной проверкой бот отдельным сообщением показывает ответы вчерашнего чек-ина, если они есть в дневнике. Если чек-ин за вчера не найден, бот покажет последний предыдущий чек-ин. Это помогает отвечать на сегодняшние вопросы с ориентиром на предыдущий день.

Если бот был выключен во время ежедневной проверки, при следующем запуске он проверит текущий дневниковый день. Если время `QUESTIONS_TIME` уже наступило, а чек-ин за этот день ещё не записан, бот автоматически запустит пропущенный опрос.

В вопросах про сон и боль можно отвечать целыми или дробными числами от 0 до 10, например `7`, `7.5` или `7,5`.

## Настройки

- `TELEGRAM_BOT_TOKEN` - токен Telegram-бота.
- `ALLOWED_CHAT_IDS` - список разрешенных чатов через запятую.
- `TIMEZONE` - часовой пояс.
- `QUESTIONS_TIME` - время ежедневных вопросов в формате `HH:MM`.
- `DIARY_DAY_START` - начало дневникового дня в формате `HH:MM`, по умолчанию `04:00`.
- `OPENAI_API_KEY` - опционально, для расшифровки голоса, описания фото и улучшенного конспекта текста.
- `GIGACHAT_API_KEY` - опционально, для конспектирования текста через GigaChat.

## Файлы

- `bot.py` - основной код Telegram-бота.
- `viewer.py` - локальный веб-просмотрщик дневника.
- `questions.json` - вопросы ежедневного чек-ина.
- `.env.example` - пример локальных настроек без секретов.
- `requirements.txt` - зависимости Python.
- `requirements-gigachat.txt` - опциональная зависимость GigaChat.
- `requirements-openai.txt` - опциональная зависимость OpenAI.
- `data/diary.jsonl` - локальный дневник, не публикуется.
- `data/uploads/` - локальные файлы вложений, не публикуются.
- `data/exports/` - локальные файлы экспорта, не публикуются.
- `data/backups/` - резервные копии дневника перед удалением записей, не публикуются.
- `data/imports/` - локальные импорты данных браслета и других источников, не публикуются.

## Хранение и несколько пользователей

Новые записи сохраняются в SQLite-базу:

```text
data/diary.sqlite3
```

SQLite встроен в Python, поэтому отдельный сервер базы данных не нужен. Если в папке `data/` уже есть старый дневник `diary.jsonl`, при первом запуске бот автоматически перенесёт его в SQLite, если база ещё пустая.

Каждая запись хранит `chat_id` и `user_id`. Команды бота `/last`, `/today`, `/day` и `/export` показывают данные только текущего Telegram-чата. Это первый шаг к режиму, где ботом могут пользоваться несколько людей.

Браузерный просмотрщик по умолчанию показывает все записи из базы. Для просмотра одного чата запустите его с ID чата:

```powershell
python viewer.py --chat-id 123456789
```

Старый JSONL можно заново выгрузить из SQLite командой:

```powershell
python storage.py
```
